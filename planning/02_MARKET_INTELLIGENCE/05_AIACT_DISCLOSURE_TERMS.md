# CARDEX — AI Act Art. 50 Disclosure: T&Cs and Privacy Notice Requirements

> **Legal basis:** EU AI Act (Reg. 2024/1689) Art. 50(2) — providers of AI systems that
> generate synthetic text published for the public must ensure each piece of output is
> marked machine-readable as AI-generated. Deadline: **2 August 2026**.
>
> **Scope:** This document specifies the required disclosure clauses for CARDEX Terms &
> Conditions and Privacy Notice, and the technical evidence trail that supports them.

---

## 1. Obligation Summary

| Art. | Text | CARDEX action |
|---|---|---|
| Art. 50(1) | Inform natural persons they are interacting with an AI system (chatbots) | N/A — CARDEX has no conversational AI in B2B interface |
| Art. 50(2) | AI-generated text intended for the public must be machine-readable marked | **REQUIRED** — vehicle descriptions generated by local LLM must carry `is_ai_generated=true` |
| Art. 50(4) | Obligations do not apply where AI output has been reviewed and modified by a human who takes editorial responsibility | Partial — dealer can edit description; system must log the override |

## 2. Required T&Cs Clause

Insert as **§ [X] — AI-Generated Content Disclosure** in the CARDEX Dealer Subscriber Agreement and in the publicly visible Terms of Use:

---

### § [X] AI-Generated Content Disclosure

**[X].1** CARDEX uses AI language models (locally hosted, no data transmitted to third parties) to generate vehicle listing descriptions where a dealer has not supplied a manual description. All such AI-generated descriptions are:

(a) marked machine-readable with `"is_ai_generated": true` in the CARDEX internal data schema, in compliance with Article 50(2) of Regulation (EU) 2024/1689 (EU AI Act);

(b) identified to end users by a visible **[AI]** badge in the CARDEX terminal and web interfaces.

**[X].2** The AI model name, generation timestamp, and a non-reversible hash of the generation prompt are stored per listing. This evidence is available to competent authorities upon request.

**[X].3** Dealers may replace any AI-generated description with a manually authored text at any time via the dealer portal. Once a manual description is submitted and saved, the `is_ai_generated` flag is set to `false` and the AI badge is removed from the listing.

**[X].4** CARDEX does not use AI to generate or modify price, mileage, year, VIN, or any other structured data field. AI generation is limited to free-text description fields only.

## 3. Required Privacy Notice Clause

Insert as a new section in the CARDEX Privacy Notice under **"How we use your vehicle data"**:

---

### AI-Assisted Description Generation

Where a vehicle listing does not include a dealer-provided description, CARDEX may use an on-premises AI language model to generate a descriptive text from the structured vehicle data (make, model, year, mileage, equipment features). This processing:

- **Legal basis:** GDPR Art. 6(1)(b) — processing necessary for the performance of the dealer subscription contract, which includes presenting a complete and marketable listing.
- **No third-party data transfer:** The language model runs on CARDEX-operated infrastructure. No vehicle data or dealer data is transmitted to external AI service providers.
- **Retention:** The AI generation metadata (model identifier, timestamp, prompt hash) is retained for the lifetime of the listing plus 90 days, to support regulatory audit and dispute resolution.
- **Your right to object:** Dealers may disable AI-assisted description generation for their account in Account Settings → Listing Defaults → "Auto-generate description". Disabling this setting does not affect existing listings.

## 4. Technical Evidence Trail

The following fields are stored in the `validation_results` table (evidence JSON) and in the `nlg_metadata` sidecar table:

| Field | Type | Purpose |
|---|---|---|
| `is_ai_generated` | bool | Machine-readable Art. 50(2) marker |
| `model` | string | Model identifier (e.g. `llama-3.2-3b-instruct`) |
| `model_version` | string | Quantisation / version (e.g. `Q4_K_M`) |
| `generated_at` | RFC 3339 timestamp | Audit timestamp |
| `prompt_hash` | SHA-256 hex (64 chars) | Non-reversible prompt fingerprint |
| `language` | BCP-47 code | Language of generated text |

All fields are validated by `nlg.AIGeneratedMetadata.Validate()` before the listing passes V11 (NLG Description Quality). A missing or invalid struct blocks publication with `SeverityCritical`.

## 6. Dealer Override Flow

```
1. Dealer submits manual description via portal PUT /listings/{id}/description
2. API validates: len(description) >= 50 chars, no generic phrases
3. On save: AIGeneratedMeta set to nil; is_ai_generated flag cleared
4. V11 re-runs on next pipeline cycle; listing re-enters publication queue
5. Audit log entry: { event: "description_override", actor: dealer_id, timestamp }
```

The override is irreversible via the UI (dealer cannot restore AI-generated text once overwritten), but CARDEX retains the original AI-generated text in the `nlg_history` table for 90 days for dispute resolution.

## 7. Competent Authority Request Procedure

Under Art. 50(2) CARDEX must be able to demonstrate compliance to market surveillance authorities (expected: national AI offices, likely Bundesnetzagentur for DE, CNIL for FR).

On receipt of a formal information request:

1. Export from `nlg_metadata` for the listed vehicle ID(s).
2. Provide: `is_ai_generated`, `model`, `generated_at`, `prompt_hash`, `language`.
3. Confirm model has not been modified or fine-tuned with personal data.
4. Provide V11 validation result confirming metadata was validated at publication time.

Response SLA: **72 hours** (align with GDPR Art. 33 breach notification cadence as operational baseline).
