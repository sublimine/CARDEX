# Germany — Plate Resolution Sources (investigation)

Probes run 2026-04-20. Responses under `/tmp/cardex_probes/de_*`.

**Structural closure**: Germany has the strictest data-protection framework
for vehicle registries in the EU. **StVG §39** and the **BDSG** (Bundes-
datenschutzgesetz) explicitly forbid public plate→owner/vehicle lookup.
Only authorised parties (law enforcement, insurance, judicial) with a
*berechtigtes Interesse* may query the **ZFZR** (Zentrales Fahrzeugregister)
operated by the KBA (Kraftfahrt-Bundesamt).

## ❌ KBA portals — no public plate interface, proxy blocks

```
curl https://www.kba.de/DE/Themen/Marktueberwachung/Rueckrufe/rueckrufe_node.html
→ HTTP 200, 342 KB marketing page. Link to Servicesuche form — searches
  by recall ID or make/model, NOT plate.

curl http://www.kba-online.de/rueckrufsuche/
→ HTTP 200 566 bytes:
  <html><head><title>Forbidden</title></head><body>
  URL http://www.kba-online.de/rueckrufsuche/ not allowed ...
  Generated by your WWW Proxy (wwwrelay) on KBA-PROXY-I.KBA.DE</body></html>
```
KBA's internal `wwwrelay` proxy blocks the historical `/rueckrufsuche/` and
`/zulassung/typ/` endpoints. The main site only exposes statistics.

## ❌ Mobilithek.info — data catalogue, SPA-only

```
curl https://mobilithek.info/
→ HTTP 200 SPA shell.
curl https://mobilithek.info/api/v1/datasets?q=kba
→ HTTP 200 but returns the same SPA HTML (SPA-routed path).
curl https://mobilithek.info/mobilithek/api/v1/metadata/search?searchTerm=rueckruf
→ HTTP 200 same SPA HTML.
```
Mobilithek is a metadata catalogue for open-data feeds, not itself a plate
lookup. Bundle analysis shows microservice names (`mdp-msa-metadata`,
`mdp-msa-files`) but no routes exposed for plate queries. Even if individual
datasets could be downloaded, none of them contain per-plate vehicle data.

## ❌ Individual stva.*.de portals (Berlin, Hamburg, München, …)

Each Landes-Zulassungsstelle exposes online forms for **appointment booking**
(Termin / Wunschkennzeichen reservation) and document upload, never a
plate→vehicle lookup. German data-protection law is explicit.

## Summary

**Germany has no public plate→vehicle service that returns data by plate.**
This is by law, not by technical oversight.

Resolver returns `ErrPlateResolutionUnavailable` with an explicit citation
of StVG §39 / BDSG. Downstream CARDEX features should prompt the user for
make+model+year directly when DE is selected, rather than attempting a
plate lookup.

### What *could* be added in a future tier (not now):
- **Recall enrichment by make+model**: if the user separately provides
  make+model, KBA recall data could be cross-referenced. But that is no
  longer a *plate* resolver — it is a separate recall feature.
- **Paid commercial resolvers** (Carfax DE, DAT, Schwacke): excluded per
  mission's zero-paid rule.
